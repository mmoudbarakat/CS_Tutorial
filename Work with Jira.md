## Work Methodologies
### Kanban
- A visual workflow system.
- Cards represent tasks moving through columns (To Do → In Progress → Done).
- Focus: Continuous delivery & limiting work-in-progress (WIP).

### Scrum
- Agile framework using fixed-length **Sprints** (1–4 weeks).
- Involves planning, daily standups, sprint reviews, and retrospectives.
- Focus: Delivering a potentially shippable increment each sprint.

___
##  Work Item Types

### Epic
- A large project goal or feature.
- Too big for a single sprint.
- Broken down into user stories, tasks, or sub-tasks.

### User Story
- A feature described from the user’s perspective.
- Format: _“As a [user], I want [function] so that [goal].”_
- Example: _“As a customer, I want to reset my password so I can regain access to my account.”_

### Task
- A standard unit of work.
- Often technical or process-related (not necessarily user-facing).
- Example: Set up CI/CD pipeline.

### Sub-Task
- A smaller unit within a Task, Bug, or User Story.
- Helps divide work into manageable parts.

### Bug
- A defect or error in the system.
- Needs to be tracked and resolved to restore correct functionality.

---

##  Planning & Management Concepts

### Backlog
- A prioritized list of all work items (stories, bugs, tasks).
- Source of truth during sprint planning.

### Priority
- Indicates urgency or business importance.
- Common levels: Highest, High, Medium, Low, Lowest.

### Component
- Represents a functional area or module of the system.
- Useful for filtering issues or assigning to specific teams.
- Examples: Frontend, Backend, Payment Service.

### Versions (Releases)
- Used to group issues related to a particular software release.
- Helps track what features or fixes are delivered when.

### Sprint
- A time-boxed iteration (usually 1–4 weeks).
- Team selects items from backlog to complete during the sprint.
- Goal: Deliver a potentially shippable product increment.
